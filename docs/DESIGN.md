# Glide 稳定设计文档

> 本文档记录当前 GUI、交互、视觉和安装体验设计事实。更新日期：2026-06-08。

## 1. 产品视觉方向

Glide 当前 GUI 使用 Rust + Slint 实现。服务端 Web 管理页使用内嵌 HTML/CSS。视觉目标是轻量、清晰、跨平台一致，不追求系统原生控件外观，也不依赖 WebView。

设计定位为 **"Developer Tool — 实时监控 + 运维控制台"** 风格：简洁、专业、克制。面向技术用户，传达连接状态、设备拓扑、错误原因和可操作建议。

## 2. 统一设计系统

### 2.1 字体系统

全局优先等宽字体，保证技术工具感、日志可读性和跨平台一致性。

| 用途 | 字体栈 |
|------|--------|
| 全局主字体 | `JetBrains Mono`, `Fira Code`, `Consolas`, `monospace` |
| 标题/导航 | 同上，`font-weight: 600/700` 区分层级 |

### 2.2 字号层级

| 层级 | 字号 | 字重 | 用途 |
|------|------|------|------|
| 页面标题 | 24px | 700 | TopBar 标题 |
| 区块标题 | 16px | 600 | Section 标题 |
| 正文/标签 | 14px | 400 | 通用文本 |
| 小字/说明 | 13px | 400 | 辅助说明 |
| 代码/日志 | 12px | 400 | 日志、设备 ID、IP |
| 徽章/标签 | 11px | 500 | 状态徽章 |

### 2.3 间距与圆角

- 间距（4px 基准）：`4 / 8 / 12 / 16 / 24 / 32px`
- 圆角：`4px`（输入框/按钮/徽章）、`6px`（卡片/Section）、`8px`（Modal）
- 边框：统一 `1px solid`
- 阴影：轻量，仅用于 Modal：`0 4px 12px rgba(0,0,0,0.15)`

## 3. 基础组件

### 已实现组件（Slint）

- `StatusBadge`：六态连接状态徽章，带呼吸动画（在线/连接中）。
- `DeviceCard`：设备名称、平台、IP、状态徽章和操作按钮。
- `SettingsRow`：带说明和 Switch 的设置行。
- `AlertBox`：四类型提示框（info/warning/success/danger），左边框 3px 标识类型。
- `EmptyState`：居中空状态：图标 + 标题 + 说明 + 可选按钮。
- `LogView`：日志视图，含顶部工具栏（复制/搜索/筛选），日志文本等宽，关键词高亮。
- `Sidebar`：页面导航，主题化背景，激活态高亮。
- `TopBar`：页面标题、连接状态徽章和主题切换按钮。

### 未单独抽象但保持设计一致的组件

- `Button`：使用 Slint 标准 `Button`，Primary/Secondary/Danger 三样式。
- `Switch`：使用 Slint 标准 `Switch`。
- `Dialog`：Modal 弹窗，遮罩层 + 内容区圆角 8px + 底部操作区。

## 5. 错误提示原则

后续错误提示必须满足：

- 说明失败发生在哪个阶段，例如地址解析、端口连接、WebSocket 握手、注册、认证、配对、权限检测。
- 给出用户可执行的下一步，例如检查服务端地址、端口、防火墙、token、权限或日志。
- 不输出 token、密钥或私钥明文。
- 不把平台限制伪装成普通失败。例如 Wayland 全局键鼠受限应明确说明。
- Windows 双击后闪退类问题需要提示用户运行 `glide.exe --smoke`、`glide.exe --diagnostics`，或查看 `%APPDATA%\Glide\logs\glide-gui.log`。
- 使用 AlertBox danger 类型展示，包含标题（失败阶段）、正文（原因）和建议操作链接。

当前 GUI 只对空服务端地址有模拟错误，真实错误提示仍需 daemon IPC / 后台服务通信接入。

## 6. 平台限制提示原则

Linux：

- X11 是第一阶段优先目标。
- Wayland 下必须显示全局键鼠能力受限，不承诺完整控制。
- 使用 AlertBox warning 类型展示合成器限制。

Windows：

- 不提示安装 WebView2。
- 键鼠共享使用 SendInput 平台后端，不安装驱动；后续如果权限或安全软件阻止输入注入，界面必须说明原因和修复方式。
- 使用 AlertBox warning 类型展示权限/安全软件限制。
- GUI 开关在 daemon 接入前仍是模拟状态，不能在文案中暗示已经完成真实跨设备控制。

macOS：

- 规划中，需要辅助功能、输入监控、剪贴板和局域网权限引导。

## 7. 安装体验原则

Windows：

- 当前是 portable zip。
- portable zip 中 `glide.exe` 和 `glide-gui.exe` 都是 Slint GUI 入口；旧 `Glide_*_x64-setup.exe` / `.msi` 不属于当前 GUI 方案。
- 后续 installer 需要保证干净 Windows 10/11 x64 可直接安装运行。
- 不能依赖用户手动安装 WebView2、Node、Rust、Python 或开发工具链。

Linux：

- 当前提供 `.deb`、`.rpm`、`.AppImage`。
- 包含 GUI、daemon、CLI、server、desktop entry 和图标。
- X11 依赖和 Wayland 限制应在文档中清楚说明。

## 8. 关键交互流程

### 8.1 主题切换

用户点击 TopBar 主题按钮 → 切换 light/dark → 全局 CSS 变量更新 → 状态持久化（Web: localStorage, GUI: 配置文件）

### 8.2 连接状态流

用户点击"连接" → StatusBadge 变为"连接中"（蓝色脉冲）→ 阶段化推进：解析地址 → TCP 连接 → WebSocket 握手 → 认证 → 同步就绪 → 任一阶段失败：StatusBadge 变为"错误"（红色）→ AlertBox danger 展示失败阶段、原因和建议操作。

### 8.3 设备配对

用户在配对页点击"开始配对" → 展示本机设备指纹 → 输入对方配对码 → 服务端验证 → 成功：设备加入信任列表 + AlertBox success → 失败：AlertBox danger 展示原因和建议操作。

### 8.4 平台限制检测

GUI 启动时检测运行环境 → Wayland 环境：PlatformPage 显示 AlertBox warning → 说明合成器限制 → 建议切换到 X11 或使用有限功能。
